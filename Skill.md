 # Pipeline-Prompt:Benchmark 比對圖雙軌分流 + Codex 交叉反驗 workflow + git push/fetch

    ## 動機(為何雙軌)                                                                                                                                                                                                                                                                
    平均場 VTK 已達數十 GB(全場 double)。login node 的 user.slice cgroup 記憶體硬上限                                                                                                                                                                                                
    ~20GB(跨使用者共用),float64 讀整顆 VTK 峰值會 OOM,且 cgroup OOM-killer 可能誤殺                                                                                                                                                                                                  
    同 slice 的別專案程序。故 benchmark 比對圖採「雙軌分流」:watcher 用省記憶體的                                                                                                                                                                                                    
    float32 inline、手動 canonical 用零誤差 float64 投計算節點。

    ## 程式碼端修改(4 處,全部只動「畫圖/腳本」層,不碰流場/solver/checkpoint)                                                                                                                                                                                                         
    ═══════════════════════════════════════════════════════════════════════════                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                                     
    ### A. `result/2.Benchmark.py` — 加 `--lowmem` 開關(預設 float64 零誤差)                                                                                                                                                                                                         
    - argparse 加 `--lowmem`(action='store_true');設 `BENCH_LOWMEM=bool(args.lowmem)`;                                                                                                                                                                                               
      `_bench_dtype = np.float32 if BENCH_LOWMEM else np.float64`。                                                                                                                                                                                                                  
    - parse_vtk 內:                                                                                                                                                                                                                                                                  
      - `_BENCH_SKIP_FIELDS = {velocity,u_inst,v_inst,w_inst,omega_u,omega_v,omega_w,P_mean}`                                                                                                                                                                                        
        (benchmark 湍流路徑全程不用的場)。                                                                                                                                                                                                                                           
      - `_bench_skip_on = (not LAMINAR)`(湍流跳過省記憶體;層流仍保留 u_inst/velocity → 兩模式都正確)。                                                                                                                                                                               
      - binary 路徑:VECTORS/SCALARS 命中 skip 時 `f.seek(npts*[3]*esize,1); f.readline(); continue`                                                                                                                                                                                  
        (跳過、不讀進記憶體)。                                                                                                                                                                                                                                                       
      - 保留場讀入改 `.astype(_bench_dtype)`(拿掉多餘 `.copy()`,astype 已回新陣列)。                                                                                                                                                                                                 
    - **不變式**:不帶 `--lowmem` ⇒ float64,與原行為 bit-identical(零誤差)。                                                                                                                                                                                                          
                                                                                                                                                                                                                                                                                     
    ### B. `watcher/hill_watcher.sh` — run_benchmark 那行加 `--lowmem`(float32 inline,0 SLURM)                                                                                                                                                                                       
    - `python3 "$BENCH_SCRIPT" --Re "$RE" --no-ask-scales --no-ask-density --lowmem`                                                                                                                                                                                                 
    - watcher **永遠 inline 在 login node 跑、不提交任何 SLURM**;頻率 = VTK-gated + FTT≥G2                                                                                                                                                                                           
      (~每 1 FTT)。float32 捨入 ~6e-8 遠低於 5% 比對精度,監控足夠。                                                                                                                                                                                                                  
    - ⚠️   running watcher 不 hot-reload → 改完要在 FTT≥G2 前**重啟 watcher**(依 heartbeat 單例規則,                                                                                                                                                                                  
      不製造重複實例)才生效。                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                     
    ### C. `lbm-plot-benchmark` 手動 shortcut → 投 dev CPU job 算 float64 最精準                                                                                                                                                                                                     
    - `result/bench_computenode.slurm`:`#SBATCH --partition=dev --gres=gpu:0 --mem=48G                                                                                                                                                                                               
      --time=00:15:00`,job 內 `cd $SLURM_SUBMIT_DIR && python3 2.Benchmark.py --Re <Re>                                                                                                                                                                                              
      --no-ask-scales --no-ask-density`(**不帶 --lowmem → float64**),用 getrusage 量峰值 RSS                                                                                                                                                                                         
      (不依賴 /usr/bin/time)。                                                                                                                                                                                                                                                       
    - 觸發 `lbm-plot-benchmark`(或手動 `python3 2.Benchmark.py`)時:                                                                                                                                                                                                                  
      1. 唯讀挑最新 stat-stable VTK;讀 FTT(< G2 標 preliminary)。                                                                                                                                                                                                                    
      2. `sbatch --test-only` 估開跑(dev 是 scavenger,可能 PENDING);再真 `sbatch`。                                                                                                                                                                                                  
      3. 背景輪詢到離隊 → sacct 終態 + log(rc + 峰值 RSS)+ ls fig_*.png。                                                                                                                                                                                                            
      4. 回報逐變數 L2(uu/vv/ww/uv/k vs Krank/Breuer)+ copy fig 到 live/。                                                                                                                                                                                                           
    - 守門:0-GPU job 不計入任何 partition 的 per-account GPU 上限(`sbatch --test-only` 實測接受);                                                                                                                                                                                    
      與主 job 只共用檔案系統、零 GPU 競爭。                                                                                                                                                                                                                                         
                                                                                                                                                                                                                                                                                     
    ### D. `.gitignore` — 忽略 benchmark job 執行期 log                                                                                                                                                                                                                              
    - `result/bench_computenode_*.log`(執行期產物,不追蹤)。

    ## 檢驗端:Codex 交叉反驗 workflow(對抗式詰問,非單票)                                                                                                                                                                                                                             
    ═══════════════════════════════════════════════════════════════════════════                                                                                                                                                                                                      
    先確認 Codex 可用(app-server 能 RC=0 啟動;若掛,先救:`~/.codex` 下壞掉的 sqlite                                                                                                                                                                                                   
    state DB,integrity_check 找壞檔→還原最新健康 .bak 或清掉讓 sqlx migrations 重建,                                                                                                                                                                                                 
    清孤兒 -wal/-shm/半殘 init 檔)。然後跑「多視角交叉反驗」:                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                     
    1. **Codex 正向審閱**(5 項 checklist):跳過場湍流未用 / `not LAMINAR` 閘 / seek 位元組數與                                                                                                                                                                                        
       對齊 / `_bench_dtype` float32 下游相容 / 拿掉 .copy() 無別名。輸出 PASS/FAIL。                                                                                                                                                                                                
    2. **對抗式詰問(workflow,≥2 個獨立 reviewer 各被指派一個「反證視角」)**:                                                                                                                                                                                                         
       - 視角1(對齊):嘗試**反證** seek 後檔案指標 desync(LOOKUP_TABLE 順序、endianness、                                                                                                                                                                                             
         esize、trailing newline)。預設「有 bug」除非能證明對齊。                                                                                                                                                                                                                    
       - 視角2(精度):嘗試**反證** float32 會改變回報的 L2 數字(找下游有無強制 float64、                                                                                                                                                                                              
         有無因 dtype 走不同分支、有無精度放大點)。                                                                                                                                                                                                                                  
       - 視角3(語義):嘗試**反證**跳過的場其實在某條路徑被用到(含 LAMINAR 分支、                                                                                                                                                                                                      
         fallback `.get()`、可視化)。                                                                                                                                                                                                                                                
       - **裁決規則**:≥多數 reviewer 反證失敗(即守住)才算該項 PASS;任一 reviewer 找到真 bug → FAIL。                                                                                                                                                                                 
    3. **經驗交叉驗證(A/B)**:同一顆 VTK 各跑 float64(dev job)與 float32(--lowmem),                                                                                                                                                                                                   
       比對逐變數 L2;差異須 ≤ ~1e-6(float32 捨入量級),否則 FAIL。                                                                                                                                                                                                                    
    4. **閘**:1+2+3 全 PASS 才放行 push;任一 FAIL → 修到 PASS,不得 push。

    ## git push + git fetch origin                                                                                                                                                                                                                                                   
    ═══════════════════════════════════════════════════════════════════════════                                                                                                                                                                                                      
    - **逐檔 stage**(禁 `git add -A`):`result/2.Benchmark.py`、`watcher/hill_watcher.sh`、                                                                                                                                                                                           
      `CLAUDE.md`(lbm-plot-benchmark 文件)、`.gitignore`、`result/bench_computenode.slurm`;                                                                                                                                                                                          
      有意義的監控產物(如 monitor_convergence_*.png)可一併。                                                                                                                                                                                                                         
    - **絕不** stage 三大紀錄檔(`Ustar_Force_record.dat*`/`timing_log.dat*`/`checkrho.dat*`,含 .gz)                                                                                                                                                                                  
      與執行期產物(`bench_computenode_*.log`、a.out*、slurm_*.log、STOP_*、*.bak.*)。                                                                                                                                                                                                
    - commit 訊息繁體中文,簡述「benchmark 雙軌分流(watcher float32 inline / 手動 float64 dev job)                                                                                                                                                                                    
      + Codex 交叉反驗 PASS」。                                                                                                                                                                                                                                                      
    - `git push`;**非 fast-forward 被拒 → 先回報讓使用者決定,不可 --force**。                                                                                                                                                                                                        
    - `git fetch origin` → 回報本地 vs `origin/<branch>` 的 ahead/behind(behind 不擅自 merge/rebase)。
